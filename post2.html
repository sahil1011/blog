Proof Of Work- Consensus methods are used in blockchain development to achieve an agreement based upon adding value in distributed node in the network. Many Blockchains have a lot of things in common and function in similar ways, but one of the ways in which blockchains can be unique is the way consensus is reached. Consensus algorithm is a process in computer science used to achieve agreement on a single data value among distributed systems. A Blockchain digital database containing some information that can be simultaneously used and shared within a large decentralized, publicly accessible network. A blocks can be mined by many techniques- 1) ASIC Mining-(Application-specific integrated circuit) it is a circuit that is customised for a specific use, it is considered the most optimized technique. 2) CPU mining-it uses a computer’s processor to validate transactions in a proof of work consensus protocol. 3) FPGA Mining — (Field-Programmable Gate Array mining)it is an circuit tailored for a specific computing task, making them several times faster at mining than normal processors. 4) Cloud Mining — it means renting their external processing power to other miners for a allotted period of time . The Production of a proof of work is a random process with low probability, so it requires a lot of trial and error on average before a valid proof of work is generated. Bitcoin’s use of a Proof of Work system is one of the defining and unique characteristics it has as a cryptocurrency. Transaction Life Cycle 1) A node requests a Transaction through a wallet. 2) Then it is broadcasted to all participation node in the blockchain network. 3) Every node in this network checks for the transaction with some validation protocols that are assigned by the head of the blockchain .These are retained into a block and are accessed using hash. Hence, becoming a part of Blockchain. Security Issues The issues related to the security of the currency are the centre of the discussion from the beginning. Huge efforts are build up to make these currency, transaction and mining secure but still many threats persist in front of this virtual currency. 1) Selfish Mining In this type of attacks selfish miners try to take on the honest miner work by wasting there computational power on wrong branch of block that is disclosed in public and keep mining the correct branch in private. This help them I earning huge profits. In general, honest miner will keep spending resources on a branch that will be terminated lately . 2) Software Attacks Most of the wallets are web based rather than desktop application .Hence chances of web based attack increases on wallet. A proper authentication is required for verification that node accessing the wallet is true. Many web based attack like phishing, dos attack leave the wallet vulnerable. 3) Double spending A double spend is an attack where the given set of coins is spent in more than one transaction. This attack can be detrimental effects to the recipients as it can affect the transaction validity. 4) Finey attack It involve pre-mine a block and spend the same coins before releasing transaction to invalidate the transaction. 5) Greater than 50% attack This involves having greater computational power of a network to reverse the transaction or less and to invalidate or validate the blocks in the chain to avoid verifying the honest transaction. 6) Time Jacks Like race attacks, attackers announces the inaccurate time stamps during the node connection. These alter the honest branch and attackers influencers only certain transactions. 7) Race attacks- These attacks includes sending both the transaction on either of honest or dishonest branch, on basis of the chances that one of the transaction gets validated.
